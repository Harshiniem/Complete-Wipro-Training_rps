What is DC Payload?

In 5G NR (New Radio), DC Payload refers to the Digital Channel Payload. This is the raw informational content (the "Transport Block") that needs to be delivered across the radio interface.

  • Not Direct Current: In this context, DC does not stand for electrical current.
  • Data Types: It can be user data (PDSCH) or control information (MIB in PBCH).
  • The Goal: The physical layer (PHY) must take these bits and transform them into electromagnetic waves that are resilient to noise, interference, and distance.

The Processing Chain

Following 3GPP TS 38.211 and 38.212, the data passes through several mathematical gates:

  1. CRC Attachment: Adds parity bits for error detection.
  2. Channel Coding: Uses LDPC (for data) or Polar (for control) to add redundancy.
  3. Rate Matching: Adjusts the coded bits to fit into the physical resource blocks.
  4. Scrambling: Randomizes bits using a Cell ID to prevent inter-cell interference.
  5. Modulation: Converts bits into complex I/Q symbols.
