PDCCH Resource Mapping

From Logical CCEs to Physical REGs (CCE-to-REG Mapping)

01 / The Hierarchy

CCE (Logical)

Control Channel Element. This is the unit used to calculate the Aggregation Level (AL).

1 CCE = 6 REGs

REG (Physical)

Resource Element Group. The smallest physical block for control mapping.

1 REG = 12 Subcarriers × 1 Symbol

Mapping Modes

Interleaved: REGs are shuffled to provide frequency diversity.
Non-Interleaved: REGs are placed sequentially.

05 / C++ Implementation

reg_mapper.cpp
/* 
 * 5G NR PDCCH CCE-to-REG Mapping Function
 * Simplified Interleaver logic
 */
#include <vector>
#include <iostream>

struct MappingResult {
    int logical_reg_idx;
    int physical_reg_idx;
};

std::vector<MappingResult> map_cce_to_reg(int agg_level, bool interleaved, int L) {
    std::vector<MappingResult> mapping;
    int total_regs = agg_level * 6; // 1 CCE = 6 REGs

    for (int i = 0; i < total_regs; i++) {
        int p_idx = i; // Default: Non-Interleaved (Sequential)

        if (interleaved) {
            // Simplified Block Interleaver Logic
            // P = j * L + (i % L) ... (Conceptual)
            int num_bundles = total_regs / L;
            int row = i / L;
            int col = i % L;
            
            // Scramble position based on bundle index
            p_idx = (col * num_bundles) + row; 
        }

        mapping.push_back({i, p_idx});
    }
    return mapping;
}
