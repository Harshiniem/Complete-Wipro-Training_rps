LDPC Base Graph Selection

Master the logic used by the 5G gNodeB and User Equipment to select the optimal Low-Density Parity-Check (LDPC) architecture for data transmission.

Understanding the Purpose

LDPC codes provide the backbone for 5G eMBB (Enhanced Mobile Broadband) data. Unlike the Turbo codes used in 4G, LDPC uses a Parity Check Matrix (H) that is extremely sparse, allowing for massive parallelization in hardware decoding.

Base Graph 1 (BG1)

Optimized for high throughput and large Transport Blocks (up to 8448 bits per code block).

Base Graph 2 (BG2)

Optimized for robustness and smaller payloads, supporting very low coding rates (down to 1/5).

System Processing Chain

Selection occurs after the Transport Block CRC is added but before Code Block Segmentation.

Selection Logic (TS 38.212)

Condition Decision Technical Context
$A \le 292$ bits Base Graph 2 Small payloads (e.g., IoT/VoLTE)
$A \le 3824$ bits AND $R \le 0.67$ Base Graph 2 Medium payloads with robust coding
$R \le 0.25$ Base Graph 2 Extreme coverage / low SNR scenarios
Otherwise Base Graph 1 Standard high-speed data flow

Hardware Implementation (C++)

// 5G NR LDPC Base Graph Selection
int getBaseGraph(uint32_t A, double R) {
    // Rule 1: Very small blocks
    if (A <= 292) return 2;

    // Rule 2: Medium blocks with low/mid rate
    if (A <= 3824 && R <= 0.67) return 2;

    // Rule 3: Very low coding rate
    if (R <= 0.25) return 2;

    // Rule 4: Default to BG1 for high speed
    return 1;
}
